# 简历优化知识库

## 简历评分标准

### 优秀简历（85-100分）
- 结构清晰，层次分明
- 使用量化数据展示成果
- 技术栈与目标岗位高度匹配
- 项目经验详细，包含STAR法则
- 无错别字和格式问题

### 良好简历（70-84分）
- 基本结构完整
- 有一定量化描述
- 技术栈基本匹配
- 项目经验较清晰

### 需要优化（60-69分）
- 结构不够清晰
- 缺少量化数据
- 描述过于笼统
- 格式需要调整

## 简历优化技巧

### 1. 使用动作动词
使用强有力的动词开头：负责、主导、开发、设计、实现、优化、提升

### 2. 量化成果
- "提升了性能" → "将接口响应时间从500ms降低到100ms，提升80%"
- "开发了功能" → "独立开发XX模块，服务10万+用户"

### 3. STAR法则
- Situation（情境）：项目背景
- Task（任务）：你的职责
- Action（行动）：具体做了什么
- Result（结果）：取得了什么成果

### 4. 技术关键词
根据岗位JD调整技术关键词：
- 后端：Spring Boot, FastAPI, MySQL, Redis, Docker, K8s
- 前端：React, Vue, TypeScript, Webpack, Vite
- AI/ML: PyTorch, TensorFlow, NLP, CV, RAG, LLM

## 常见简历问题

### 问题1：工作经历空洞
**错误写法**：负责后端开发
**优化写法**：负责核心交易系统后端开发，采用微服务架构，日均处理订单50万+，P99延迟<200ms

### 问题2：技能罗列无重点
**建议**：按熟练程度分类，突出与目标岗位相关的技能

### 问题3：项目经验重复
**建议**：每个项目突出不同侧重点和技术亮点
