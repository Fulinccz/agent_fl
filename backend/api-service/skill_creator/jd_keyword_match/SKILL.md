# JD 关键词匹配技能 (JDKeywordMatchSkill)

## 作用
分析招聘 JD 关键词，自动优化简历以提高关键词匹配度，提升 ATS（简历筛选系统）通过率。

## 输入
- **简历原文**：用户的完整简历或工作经历
- **JD**：目标岗位的招聘描述

## 执行流程

### 1. 本地 RAG 提取 JD 关键词
**关键词分类提取：**

#### 技术关键词
- 编程语言：Python、Java、Go、C++ 等
- 框架：Spring Boot、Django、React、Vue 等
- 工具：Docker、Kubernetes、Git、Jenkins 等
- 数据库：MySQL、Redis、MongoDB、Elasticsearch 等
- 云平台：AWS、Azure、阿里云、腾讯云等

#### 能力关键词
- 软技能：团队协作、沟通能力、领导力等
- 方法论：敏捷开发、Scrum、DevOps 等
- 业务：高并发、分布式、微服务等

#### 职责关键词
- 常见动词：设计、开发、优化、主导、推动等
- 职责描述：系统设计、架构规划、性能优化等

### 2. 关键词匹配度分析
**计算当前匹配度：**
```python
匹配度 = (简历中出现的 JD 关键词数 / JD 总关键词数) × 100%
```

**识别缺失关键词：**
- JD 要求但简历未提及的技能
- JD 强调但简历弱化的经验

### 3. 自动插入到经历中
**插入策略：**

#### 自然融入
```
原始：使用 Python 开发后端接口
JD 关键词：FastAPI、微服务、高并发
优化：基于 FastAPI 开发高并发微服务接口
```

#### 技能补充
```
原始：负责系统开发
JD 关键词：分布式、Kubernetes、AWS
优化：负责分布式系统开发，基于 Kubernetes 在 AWS 上部署和管理
```

#### 项目增强
```
原始：做了个电商平台
JD 关键词：高可用、Redis、MySQL 分库分表
优化：主导高可用电商平台开发，使用 Redis 缓存和 MySQL 分库分表方案
```

### 4. 关键词密度优化
**原则：**
- 核心关键词出现 2-3 次
- 避免过度堆砌（<5% 密度）
- 分布在不同模块（技能栏、项目经历、工作经历）

## 输出
关键词增强版简历，包含：
- ✅ 优化后的简历内容
- ✅ 关键词匹配度报告（优化前 vs 优化后）
- ✅ 插入的关键词列表
- ✅ ATS 通过率预估

## 实现要点

1. **关键词提取**：使用 RAG + NLP 技术从 JD 提取关键词
2. **匹配度计算**：计算简历与 JD 的关键词重合度
3. **智能插入**：在保持语义通顺的前提下自然融入关键词
4. **密度控制**：避免关键词堆砌，保持自然度
5. **真实性校验**：确保插入的关键词与用户经历匹配

## 注意事项

- ⚠️ 不虚构用户没有的技能
- ⚠️ 关键词插入要自然，不生硬
- ⚠️ 保持简历真实性
- ⚠️ 过度优化的关键词可能被面试官识破
